## 一、这是什么

给一个将满 11 岁、踢球的男孩（乔飞）做的足球"球商"训练工具。核心痛点：**传完球爱站着不动、场上观察意识差**。不是技术训练，是**用脑子**——让他在战术板上自己摆跑位、说想法，把"该跑该看"从"知道"变成"本能"。

爸爸是执行者（业余球员，懂球），AI 负责每天出场景 + 复盘。这个 HTML 是父子俩在 iPad 上用的战术板：摆位、画轨迹、攒"我能想出几种跑法"、记录、攒进度。

完整背景见项目母文档《罗乔飞球商提升计划 v1.0》（爸爸手里），这里只讲战术板本身。

## 二、最重要的设计哲学（红线，别破）

这套东西最容易被"做成考核机"。之前有一版 v2.0 升级方案差点把它改成 KPI 系统（自动解锁、每日自评分、每招评分、成绩单），被否掉了。守住这几条：

1. **一把尺子**：任何新功能问一句——"它是让他**更想动脑+更想玩**，还是给了他一个**可以去优化的数字**？前者留，后者砍。"
2. **通关由爸爸判断，不由公式判断**。段位是手动升的，不做"攒够X星自动解锁"。一旦分数变成目标，孩子就去刷指标、揣摩大人口味，探索性就死了（Goodhart定律）。
3. **不锁内容**。所有场景永远可选，不把"酷"的中锋场景锁起来逼他通关——对一个踢得不错又急性子的孩子，锁会憋出抵触甚至恨足球。
4. **不做每日自评分进系统**。让嘴硬好胜的 11 岁每天给自己打分，要么虚高要么别扭，还多一个能刷的数字。（口头偶尔"你先猜分我再说分、看差距"可以，但不进代码。）
5. **先上线、跑一周、看真实反馈再加功能**，别先把功能堆满。原则：方法服务于孩子，不是反过来。

## 三、当前已实现（v1.3）

### 四个标签页
- **今日**：当前场景（情境+问题）→ 战术板摆位 → "攒招"（记下多种跑法）→ 他的答案 + 家长打星 → 保存
- **历史**：累计天数/连续天数/累计星星 + 每日记录卡 + 【导出/导入备份】按钮
- **场景库**：14 个场景，按段位分组，可切换；可新建自定义场景
- **⚔️ 挑战**：他出题摆个难球考爸爸，爸爸答，他给爸爸打分（"挑战爸爸"——出题本身就是高强度动脑）

### 段位
青铜/白银/黄金/王者，右上角下拉**手动切**（不自动解锁，见红线2）。

### 场景库（14个，BUILTIN 数组）
- 青铜6：传完之后那一下 / X1传完3秒 / W2球被倒回来 / W1边路1对1 / 传完反插 / 队友丢球了
- 白银5：X2接球前先看 / 弱侧偷袭 / 接球前扫视 / F3中锋背身做墙 / 中锋抢点时机
- 黄金3：中锋背身护球 / 反击2打2 / 做墙后跑位
- 王者0（集训后再加，现在投入太早）
- 每个场景含：rank/title/sit(情境)/q(问题)/twists(变招提示数组)/tokens(初始棋子位置)

### 关键交互（v1.5 —— 拖动产生线 + 球吸附，取消画笔）
- **核心交互**：没有画笔，**直接拖球员或球，线型自动判定**：
  - 拖球员（脚下无球）→ 跑动（实线）
  - 拖球员（脚下有球，球粘着他）→ 带球（波浪线），球跟着脚一起走
  - 拖球（从持球人拖给队友）→ 传球（虚线），松手时球自动吸附到最近球员脚下
- **球吸附**：球离某球员 <8% 即"粘"在他脚下（holderOf/syncBallToHolder）。下次拖那人就是带球。这是真实足球直觉，乔飞一看就懂。
- 每次拖动结束=一笔；多笔串起来打进球 → 点"⚽这套打进了！"存为一种解法；攒"几套不同解法"= 发散思维。
- 拖的**整条轨迹**就是线（带中点贝塞尔平滑），保留他"特意绕后点/直插"的意图。
- 功能区（撤销/打进了/复位/加人/删人）紧贴球场下方，手不用往下够。
- **变招提示**：默认收起，点"💡 想要提示？"才展开。
- **数据捕获**：保存时每套解法的每一笔存 type(run/pass/dribble)+完整路径点+顺序。导出 JSON 后能完整复现他的思路。

> v1.4 是"选画笔再描线"（分离式），v1.5 改成"拖谁画谁"（沉浸式），用户反馈分离太麻烦。chPitch（挑战页）仍用旧的 set/tokens/add/edit API，不受影响。

## 四、改进过程（按时间，含每步原因）

起点是 Claude 之前做的 `qiaofei-tactics-board_1.html`（48K，725行）。本轮在它基础上改进，逐步到 v1.3。

| 步骤 | 改动 | 为什么 |
|------|------|--------|
| 1 | **存储层加 localStorage 双写** | 原版只认 `window.storage`（Claude Artifact 专有 API）。下载到本地/iPad 用浏览器打开时它不存在，数据只在内存，刷新就丢。这是"跑一周看反馈"的前提，最高优先级。改成三层降级：window.storage → localStorage → 内存。 |
| 2 | **变招提示改收起式** | 原版一加载就铺开所有提示。改成默认收起+"想要提示？"按钮，符合"先自己想"的心法。 |
| 3 | **场景 6→14 个** | 原版只 6 个撑不过一周。按痛点扩到 14（传完不动/观察/中锋方向为主）。**注意：扩场景时中文双引号被误写成英文引号，导致 JS 字符串提前闭合、整个 script 崩、全页面点不动——这是个大坑，改完必须做语法校验。** |
| 4 | **PWA 化** | 加 apple-mobile-web-app meta + apple-touch-icon。iPad"添加到主屏幕"后像真 App、全屏、对好胜心加分。 |
| 5 | **图标修复（折腾了3次）** | data:URL 图标 iOS 不认（显示成"乔"字）→ 改真实 PNG；qlmanage 转的 PNG 星跑到左上角 → 改用 Pillow 逐像素画居中金色★铺满画布。 |
| 6 | **轨迹三线型升级（v1.3 核心）** | 原版所有移动都是同一种彩虚线，跑动和传球分不清，球还根本不画线。改成战术板国际通用线型语言：**实线=跑动 / 虚线=传球 / 波浪线=带球**，颜色仍分你/队友/对手；路线改平滑贝塞尔曲线；自动判定带球（球员+球一起移动→波浪线并抑制重复球线）；图例加说明。教育价值：乔飞从小用对符号=顺带学会看懂专业战术图。 |
| 7 | **导出/导入备份** | localStorage 是单设备本地存档，清缓存/换设备会丢。历史页加两按钮：导出全部数据成 JSON 文件、导入恢复（带格式校验+覆盖确认）。用于换设备恢复、发给 AI 做月度复盘。（注：用户本想"保存直接上传 git"，但 public 静态页嵌 token 会暴露，安全红线，所以先做手动导出；真要自动云同步得上 Cloudflare Worker 中转。） |
| 8 (v1.4) | **引擎重做：手指描线 + 完整解法 + 进球闭环 + 富数据** | 用户反馈三点：①闭环不清晰，要从开局到进球的完整链路再想下一种；②轨迹是起点→终点直线，画不出真实跑动/带球路线；③希望从"摆的过程"看到他思路、给数据能复盘。**全部在 Pitch 控制器重做解决**：拖棋子→手指描线；单点位置→多笔序列；一招→一套通往进球的完整解法；存档从"几招"升级到"每笔的类型+完整路径点+顺序"。这是从"快照"到"完整解法"的范式升级，是本项目目前最大的一次改动。 |

## 五、关键代码位置（index.html，v1.4 后行号有变，用函数名定位）

- 存储层 `store` 对象：三层降级 getJSON/setJSON（window.storage→localStorage→内存）
- 存储 keys `var K`：custom/sessions/progress/current/ch
- 场景库 `BUILTIN` 数组：14 个场景
- `state` 对象、`Pitch(boardId,trailId)` 共享战术板控制器（v1.4 已重写）
- **Pitch 控制器内部（v1.4 引擎核心）**：
  - 状态：`tokens`(棋子) / `pen`(当前画笔 null|run|pass|dribble) / `strokes`(当前这套已画的笔) / `solutions`(已存的完整解法) / `drawing`(正在描的笔)
  - **`drawTrails()`**：渲染已存解法(淡显)+当前笔(实显带步号)+正在画的笔(跟手)
  - **`smoothPath(pts)`**：手指描的多点路径→平滑折线(中点二次贝塞尔)
  - **`wavyAlong(pts)`**：沿描线路径生成波浪线(带球)，按弧长重采样+垂直正弦
  - **`strokeStyle(p,type)`**：按 run/pass/dribble 上色和线型
  - board 的 `pointerdown` 监听：pen 非空时接管描线
  - 对外 API：`setPen/getPen/strokeCount/undoStroke/commitSolution/solutionCount/solutions/clearAll/onChange`，旧 API `set/tokens/add/edit/reset/draw` 保留（挑战页 chPitch 仍用）
- `setPenUI(pen)`：画笔按钮高亮 + 提示文案
- `updateOptUI()`：解法计数 + 鼓励多套的话术
- `saveSession()`：保存当天，**富数据在这**（solutions 数组：每套 steps，每笔 type+points）
- `bind()`：事件绑定（画笔切换、撤销、进球、复位、导出/导入）
- SVG 箭头 marker 定义（含 `ar-ball`）：约 234 行
- 导出 JSON 结构：`{_app:"qiaofei-tactics-board",_version:"1.4",_exportedAt,custom,sessions,challenges,progress,current}`
- session 结构（v1.4）：`{id,date,scn,rank,stars,solutions:[{steps:[{type,points:[{x,y}]}]}],solCount,ans,note}`

坐标系：所有坐标用百分比（x,y ∈ 0-100）。画笔路径 `points` 是百分比点数组。战术板 `aspect-ratio:3/4` 竖屏，进攻方向向上。注意 v1.4 后 token 不再有 x0/y0（起点），轨迹改由独立的 strokes 记录。

## 六、待办（按优先级，都属"跑一周后看真实反馈再加"的第二批）

**最值的两条（用户和评审都认）：**
1. **实战连接**：场景库加"赛后实战"入口，但**别做成"复盘你的表现"**（他叛逆抗拒），做成"把今天比赛那个球摆出来考考爸爸"——直接挂到「挑战爸爸」上，他以为在为难爸爸，其实在做迁移。历史里给绿色"实战"badge。
2. **挑战难度标签**：出题时选 🥉简单/🥈正常/🥇必杀技，统计分开。他会冲着出"必杀技"，为了出难题得先想清楚什么球难、有几种解——本身就是球商。

**第三批：**
3. 关卡庆祝仪式：接在**爸爸手动升段**那一下（挣来的勋章，不是公式自动发），动画克制别花哨。
4. 进步曲线：只做给**爸爸复盘**用，画频率/连续天数这类不好刷的，**别画星星**（画星=又把星变 KPI）。

**砍掉/无限期搁置（违反红线，别做）：**
- 每招"把握度"评分、每日自评分进系统、成绩单发朋友圈/给外部教练看（把成长变成给外人看的成绩单=加压，违反"不和哥哥比、别让他证明给别人看"）。

**技术债：**
- 自动云同步（Cloudflare Worker 中转 + 细粒度 token），等手动导出不够用了再上。
- 改任何含中文标点的 JS，**务必先做语法校验**（`node -e "new Function(scriptText)"`），中文引号/英文引号混用是踩过的坑。
